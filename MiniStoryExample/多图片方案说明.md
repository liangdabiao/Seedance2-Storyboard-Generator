# Seedance 1.5 Pro 多图片输入方案

## 问题背景
原始方案只使用1张参考图片，无法充分利用每集配置中的多张素材图片（角色、场景、道具等）。

## API限制发现

通过多次测试，发现Seedance 1.5 Pro API的图片输入限制：

### ❌ 不支持的方案
1. **多张无role图片**: "expected at most one image content with unspecified role"
2. **多张reference_image**: r2v模式不支持1.5 Pro模型
3. **first_frame + 普通图 + last_frame**: "last frame cannot be mixed with first frame or reference image"

### ✅ 支持的方案
1. **1张普通图**: 不带role参数
2. **first_frame + 普通图**: 首帧+1张参考图
3. **first_frame + last_frame**: 首帧+尾帧（最佳方案）

## 最终采用方案

### 策略
```python
if 图片数量 == 1:
    使用普通参考图（无role）
    
elif 图片数量 >= 2:
    第1张 → first_frame（首帧）
    最后1张 → last_frame（尾帧）
    中间图片 → 跳过（记录日志）
```

### 优势
1. **最大化利用配置**: 使用首尾2张图片而非只用1张
2. **更好的视频连贯性**: 首尾帧可以指导视频的起始和结束画面
3. **符合API限制**: 完全符合Seedance 1.5 Pro的要求

### 实现示例

#### E01配置（5张图）
```json
{
  "id": "E01",
  "images": ["C01", "S01", "S02", "P01", "P03"]
}
```

**使用方式**:
- 首帧: C01 (小兔子角色)
- 尾帧: P03 (茉莉花手环)
- 跳过: S01, S02, P01

#### API请求
```json
{
  "content": [
    {"type": "text", "text": "提示词..."},
    {
      "type": "image_url",
      "image_url": {"url": "https://.../C01.png"},
      "role": "first_frame"
    },
    {
      "type": "image_url", 
      "image_url": {"url": "https://.../P03.png"},
      "role": "last_frame"
    }
  ]
}
```

## 注意事项

1. **中间图片会被跳过**: 由于API限制，只能使用首尾2张图片
2. **合理安排素材**: 建议将最重要的角色/场景放在首尾位置
3. **提示词补充**: 可以在prompt中描述中间图片的内容来弥补

## 效果预期

使用首尾帧后，视频生成时：
- **首帧**: 视频开始画面会参考第1张图片
- **尾帧**: 视频结束画面会参考最后1张图片
- **中间过渡**: 由AI根据提示词自动生成

这样可以获得更连贯、更符合预期的视频效果。
